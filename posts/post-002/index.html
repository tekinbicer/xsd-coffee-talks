Recent performance improvements to TomoPy

Posted at — Oct 21, 2019

Abstract

We will present recent performance changes related to TomoPy, an open-source Python package for tomographic data processing and image reconstruction. Notably, we have added a number of new reconstruction methods, added new benchmarking tools, and improved reconstruction speeds for various reconstruction algorithms by targeting Nvidia GPUs and specific supercomputing architectures.

Starting in 2017, through the NERSC Exascale Science Applications Program (NESAP), we started updating the code-base to take advantage of modern compilers such as Clang and the Intel compiler; to target specific super-computing CPU architectures such as Edison, Haswell, and Knights Landing; and to utilize Nvidia GPUs through CUDA for certain tasks. By using a rotation-based and log-polar based implementations of forward and backward kernels on the GPU, previously determined to be inefficient on the CPU, TomoPy has seen computational speed improvements that exceed 500x. On HPC systems at NERSC, runtimes for 50 iterations on a single slice using iterative reconstruction algorithms such as SIRT or MLEM have dropped from approximately 6.5 hours down to approximately 40 seconds. In this presentation, we will compare the reconstruction speed and accuracy of the older implementations (TomoPy v1.1) with our performance enhanced implementations and the following newly implemented reconstruction algorithms: expectation maximization, total variation regularization, gradient descent and conjugate gradient descent.

This work is a collaboration between Jonathan Madsen (NERSC), Viktor Nikitin (Max IV), and Doga Gursoy (APS).
